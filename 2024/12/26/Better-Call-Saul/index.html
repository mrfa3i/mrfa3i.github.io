Solution

In static analysis, here we can see that the mobile application was made in native cpp library. that means your Java or Kotlin code can then call functions in your native library through the Java Native Interface (JNI).
You can read more about the native library here.

There are two functions

  1. MainActivity
  2. Secret

Now let‚Äôs run the mobile application using Android Studio to see what it does.

Here we can see that the mobile app has a button labeled ‚ÄúCall me!‚Äù but when you press on it, nothing happens, so based on the challenge description, we conclude that we want to call something, and here there is a Secret function. Let‚Äôs try to call it.

To call another function in mobile apps, the exported function must be true.

In AndroidManifest.xml the export function set false

Note: The manifest file describes essential information about your app

Let's try to patch the application and set the ``exported`` value to ``true``

  1. Decompile the APK file.

  2. Open AndroidManifest.xml in notepad and edit exported=false to true

  3. Rebuild

  4. Sign your app
    Android requires that all APK files be digitally signed with a certificate before they are installed on a device or updated.

    To patch an APK, you must first modify it as needed, then re-sign it with a valid certificate. You can easily find instructions on signing APKs by searching online for "how to sign APK keys."

    Now install ``aligned.apk`` in the device and just call secret
